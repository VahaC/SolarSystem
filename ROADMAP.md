# 🛰️ Roadmap

Ideas for future versions of the Solar System simulation, grouped by impact-vs-effort.
Priorities are subjective — anything here is fair game, in any order.

---

## 🥇 Top picks (highest wow-per-line-of-code)

1. ~~**Bloom / HDR glow on Sun & flares** — fullscreen post-process: extract bright pixels → Gaussian blur → additive composite. Will instantly raise the perceived production value.~~ ✅ **Done** — implemented in `Renderer` as an HDR (RGBA16F) offscreen target + bright-pass + 6-pass separable Gaussian blur + additive composite (`BeginScene` / `EndSceneAndApplyBloom`).
2. ~~**Smooth focus transition + planet trails** — lerp `Camera.Target` and `Distance` over ~0.5 s instead of snapping; render a fading line strip behind each planet for the last *N* positions. Looks "cinematic" especially at high simulation speed.~~ ✅ **Done** — `SolarSystemWindow.FocusOn` now starts a 0.5 s smoothstep lerp on `Camera.Target`/`Distance` (tracking the body's live position en route); each `Planet` keeps a 200-sample ring buffer that `Renderer.DrawTrails` rasterises as a per-vertex-alpha-fading `LineStrip`. Toggle with `T`.
3. ~~**Logarithmic depth + minimum screen-size dots** — make the real-scale (R) mode actually usable: log-depth shader (`gl_FragDepth = log2(1+w)/log2(1+far)`) eliminates z-fighting at huge near/far ratios, and a per-body screen-space minimum size (e.g. ≥ 2 px) keeps planets visible even from astronomical distances.~~ ✅ **Done** — every 3D shader (`PlanetVS`/`SunVS`, `OrbitVS`, `RingVS`, `GlowVS`, `TrailVS`, `SolarWind`, `SolarFlares`) now writes `gl_Position.z = (log2(1 + w) * Fcoef - 1) * w` with `Fcoef = 2 / log2(far + 1)`, eliminating z-fighting across the 10⁷ near/far ratio of real-scale mode. `PlanetVS` additionally enforces a screen-space minimum silhouette by radially expanding vertices outward from `uPlanetCenter` whenever the body's projected radius drops below `Renderer.MinPixelRadius` (1 px → ~2 px diameter dot), so planets stay visible at any distance.

## 🎨 Visual polish

| # | Feature | Notes |
|---|---|---|
| V1 | Bloom / HDR glow on Sun & particles | ✅ Implemented (RGBA16F FBO + bright-pass + separable Gaussian + additive composite). |
| V2 | Atmospheric rim-light (Earth, Venus, Jupiter, Neptune) | Fresnel `pow(1-dot(N,V), 3)` tinted by atmosphere colour in planet shader. |
| V3 | Earth cloud layer | ✅ Done. `Renderer.DrawClouds` rasterises a second sphere at `1.012 × VisualRadius` using a dedicated `CloudFS` (reuses `PlanetVS` so log-depth + min-pixel apply). Alpha is derived from cloud-texture luminance via `smoothstep(0.18, 0.92, lum)` so the JPG source works without a dedicated alpha channel. Drawn alpha-blended after the opaque planet pass with depth-write off. The cloud sphere has its own `Planet.CloudRotationAngleRad`, advanced at `(spinRate − 0.08 rev/sim-day)` so the layer drifts westward relative to the surface. Loads `textures/8k_earth_clouds.jpg` opportunistically — missing file → no clouds. |
| V4 | Earth night-side city lights | ✅ Done. `PlanetFS` now takes an optional `uNightTex` + `uHasNight` flag; on the dark side of the terminator it adds `night × (1 − smoothstep(-0.05, 0.2, NdotL))` to the lit colour, so emissive city lights only appear where direct sun illumination drops off. `Planet.NightTextureId` is populated from `textures/8k_earth_nightmap.jpg` when present (no-op otherwise), and the night sampler is bound on TU1 in `Renderer.DrawPlanet`. |
| V5 | Saturn ring shadow on the planet (and vice versa) | ✅ Done. `PlanetFS` now takes `uHasRing/uRingInner/uRingOuter/uRingNormal/uRingTex`; for fragments on the lit hemisphere of Saturn it casts a ray toward the Sun, intersects the ring plane (normal = `RotZ(tilt)·(0,1,0)`), and if the hit point falls between the inner/outer ring radii samples the ring texture's alpha and attenuates the diffuse + specular terms by `1 - α`. `RingFS` ray-marches the inverse: from each ring fragment toward the Sun it solves a ray-vs-sphere against Saturn (`uPlanetCenter`, `uPlanetRadius`), darkening the fragment to 25% when the segment hits the planet. `RingVS` was extended to output `vWorldPos` and `Renderer.DrawPlanet` now binds the ring texture on TU2 + uploads ring uniforms only for `Saturn`. |
| V6 | Lens flare on the Sun | ✅ Done. New `Renderer.DrawLensFlare` projects the Sun's world position to NDC (skips when behind camera, faded when off-screen), then renders a fullscreen additive quad whose `LensFlareFS` synthesises 6 coloured ghosts at fixed fractions along the sun-through-screen-centre axis with aspect-corrected radii. Intensity is gated by `dot(viewForward, dirToSun)` so the ghosts only ignite when the Sun is roughly looked-at. Drawn after `EndSceneAndApplyBloom` so it composites on top of the bloomed scene without being smeared by the Gaussian blur. |
| V7 | Improved Milky Way sky | ✅ Done. `SkyFS` reconstructs a per-pixel world-space view direction (clip→`uInvViewProj`) and samples `_starsTexture` as an equirectangular map (`atan2(z,x)`/`asin(y)`) on a fullscreen quad at the far plane, so the procedural noise is replaced wholesale by the real Milky Way panorama whenever a texture is present. `Renderer.Initialize` opportunistically loads `textures/8k_stars_milky_way.jpg` (current convention) with a fallback to `textures/8k_stars_milkyway.jpg` (roadmap spelling); a tiny solid-colour procedural texture is kept only as a last-resort fallback when neither file ships. |
| V8 | Eclipses & body shadows | ✅ Done. `PlanetFS` now takes `uShadowCount` + `uShadowSpheres[16]` (xyz=center, w=radius). Per fragment it casts a ray from the surface toward the Sun and tests every caster with a `dot(oc, ldir) / |oc - ldir·tca|²` cylinder check; if any sphere intersects the segment its silhouette feeds a `smoothstep(r·0.85, r, d)` soft-edge term that attenuates `diff + spec`. The shaded body is skipped via center-match against `uPlanetCenter`. `Renderer.SetShadowCasters` uploads the array once per frame from `SolarSystemWindow.BuildShadowCasters` (Moon + Galileans + Titan + planets, capped at 16). The same eclipse term also gates the V9 atmosphere, so a planet in totality goes properly dark. |
| V9 | Atmospheric scattering | ✅ Done. `PlanetFS` adds a `uHasAtmosphere/uAtmosphereColor/uAtmosphereStrength` branch on top of V2's Fresnel rim: a `pow(1-NdotV, 3)` rim is modulated by Rayleigh `0.75·(1+μ²)` and Henyey-Greenstein Mie (`g=0.76`) phase functions, with a soft terminator wrap (`NdotL + 0.25`). `Renderer.GetAtmosphere` ships per-body coefficients for Earth (cool blue), Mars (rust), Venus (sulphur), Titan (orange haze) and Neptune (deep blue). The whole term is multiplied by the V8 eclipse factor so an eclipsed planet's atmosphere also dims. |
| V10 | ACES tone mapping + exposure | ✅ Done. `composite.frag` now applies Krzysztof Narkowicz's ACES filmic curve to `(scene + bloom·strength) · uExposure`, replacing the bare additive clamp. `Renderer.Exposure` sets the manual stop. With `Renderer.AutoExposureEnabled = true` the HDR scene texture is allocated mip-complete (`LinearMipmapLinear`) and `glGenerateMipmap` runs each frame between the bloom blur and the composite; the shader then samples `textureLod(uScene, vec2(0.5), 20.0)` (the 1×1 mip) for average luminance and corrects exposure toward middle-grey 0.18 with a 0.25–4.0 clamp so deep-space frames don't over-correct. |
| V11 | FXAA / TAA on the post pass | ✅ Done. New `fxaa.frag.glsl` (compact FXAA 3.x: 5-tap luma neighbourhood → 0.0312 / 12.5 % contrast threshold → directional 4-tap blur with luma-range fallback) renders from a new RGBA8 LDR target into the default framebuffer. The composite pass is rerouted to that LDR target whenever `Renderer.FxaaEnabled` is on (default true); when disabled the composite goes straight to the default FB as before. Allocated alongside the existing HDR/bloom targets in `EnsurePostProcessTargets`. |
| V12 | Sun corona / surface granulation | ✅ Done. `sun.frag.glsl` now takes `uTime` + `uCorona`; when on, a 4-octave fbm of `normalize(vNormal)*4 + (0, uTime*0.04, 0)` drives a per-fragment pulse (`0.85 + 0.45 n + 0.15 sin(uTime*0.7 + n*9)`) and a `mix(cool, hot, smoothstep(0.35, 0.75, n))` tint, so granules ripple across the disc and never align with the texture seam. `Renderer.DrawSun` uploads `uTime = (float)GLFW.GetTime()` and `uCorona`. Toggle with `U` (`Renderer.CoronaEnabled`, persisted). |
| V13 | Aurora at Earth & Jupiter poles | ✅ Done. New `Aurora` owns a 128-segment triangle-strip ribbon (top edge = bright, bottom edge = transparent); `aurora.vert.glsl` reconstructs each vertex on the unit sphere from `(angle, side)` between latitude bands 72°–80°, modulated by `sin(angle*6 + t*0.8) + sin(angle*13 - t*1.3)` curtain waves; the same VBO renders both poles via `uHemisphere = ±1`. `aurora.frag.glsl` adds `0.55 + 0.30 sin(angle*9 + t*1.2) + 0.15 sin(angle*23 - t*2.1)` shimmer and writes additive RGBA so the ribbons feed the bloom pass. `SolarSystemWindow` calls `Aurora.DrawForBody` for Earth (cool green) and Jupiter (magenta/violet) inside the HDR window; intensity halves when the solar wind is off. Toggle with `K` (persisted). |
| V14 | PBR planet shading | ✅ Done. `planet.frag.glsl` now branches on `uPbrEnabled`: when on, it runs Cook-Torrance with GGX normal-distribution `D = α² / (π d²)`, Schlick-GGX geometry `G` with `k = (r+1)²/8`, and Schlick Fresnel `F` over `F0 = mix(0.04, base, uMetallic)`; the diffuse term is energy-conserving via `kd = (1-F)(1-uMetallic)`. `Renderer.GetPbr` ships per-body roughness/metallic constants (rocky 0.80–0.95, gas-giant 0.40–0.65, Earth 0.45). The legacy Phong lobe stays available as the `uPbrEnabled = 0` fallback. Toggle with `I` (`Renderer.PbrEnabled`, persisted). |
| V15 | Specular ocean mask for Earth | ✅ Done. New `Planet.OceanMaskTextureId` is loaded opportunistically from `textures/8k_earth_specular_map.{png,jpg,tif}` for Earth only. `Renderer.DrawPlanet` binds it on TU3 and feeds `uOceanMask` + `uHasOceanMask` to `planet.frag.glsl`, which multiplies the specular term (both Phong and PBR branches) by the mask’s red channel so only oceans glint and continents stay matte. Missing texture → no-op (legacy uniform glint). Toggle with `Q` (`Renderer.OceanMaskEnabled`, persisted). |

| S8 | Constellation overlay | ✅ Done. `Constellations` loads RA/Dec line endpoints from `data/constellations.json` and renders them skybox-style (translation stripped from the view matrix + `gl_Position.z = w` so the figures sit on the celestial sphere at infinity, independent of camera position or scale mode). | S10 | Lagrange points | ✅ Done. Renders each as a small additive diamond + label (orbit-line shader). Toggle with `G`. |
| S11 | Meteor showers | ✅ Done. New `MeteorShowers` ships seven peaks (Quadrantids, Lyrids, Eta Aquariids, Perseids, Orionids, Leonids, Geminids) as `(month, day, RA/Dec radiant, rate)`. When sim time falls within ±3 days of a peak it emits short additive streaks travelling away from the radiant, born on a disk perpendicular to it just sunward of Earth, using A4's instanced-quad pipeline and A5's fixed-substep integration. | S12 | Eclipse / transit calendar | ✅ Done. New `Bookmarks` ships a hand-curated list of notable Sun–Earth–Moon alignments and Mercury / Venus transits (1999–2045) as `(title, UTC date, kind)`. `Ctrl+B` cycles forward to the next bookmark whose date is past the current `simDays` (wraps to the first), snaps `_simDays` to it, clears trails so they don't draw a stale arc across the new epoch and shows a banner with the event title. |
| S13 | Tidal locking visualisation | ✅ Done. New `TidalLock` enumerates every spin-locked moon (Earth's Moon, the Galileans, Titan) and rasterises a small additive 3-segment arrow (shaft + arrowhead) on each one's near-side hub-axis pointing at its host planet, regenerated every frame so it tracks both the moon and the host's motion. Toggle with `F4`; persisted via `PersistedState.ShowTidalLock`. Localised banner key `ui.tidal.on/off`. |
| S14 | Planetary alignment indicator | ✅ Done. New `PlanetaryAlignment` computes each major planet's heliocentric ecliptic longitude per frame, runs union-find over pairwise angular distance with a 12° threshold and reports every component of size ≥3 as a `Group(indices, names)`. The drawer rasterises an additive line chained from the Sun through every member out to a ray-tip beyond the outermost body, and a top-right banner names the participants (`ui.alignment.banner = "Alignment ({0}): {1}"`). Toggle with `F5`. |
| S15 | N-body perturbation mode | ✅ Done. New `NBodyIntegrator` runs a velocity-Verlet (kick-drift-kick leapfrog) on the major planets in (AU, days, M⊙) units with `GM_sun = 4π²/365.25² ≈ 2.959e-4 AU³/d²`. The Sun is fixed at the origin; mutual gravity uses each planet's tabulated mass in solar masses (Mercury…Neptune). On toggle / date jump / scrubber drag the integrator resyncs from the analytic Kepler position + a 1-day central finite-difference velocity. Adaptive sub-stepping caps `|dt|` at 0.5 d/step (200 steps/frame max). Toggle with `F6`; bottom-right “N-body mode” banner stays up while it's active; persisted via `PersistedState.NBodyEnabled`. Dwarfs / moons / comets stay analytic so the contrast against the perturbed majors is the whole point of the feature. |
| S16 | Real comet catalogue | ✅ Done. `Comet`'s constructor now takes an externally-supplied `Planet` nucleus, and the new `CometCatalog` loads a JSON catalogue (`data/comets.json`, comments + trailing commas tolerated) of well-known comets — Halley, Hale–Bopp, NEOWISE, Encke — each with full Keplerian elements + per-comet tail tuning (`emissionRate`, `tailLifetime`, `tailSpeed`). New `Comets` plural manager owns the array and exposes the same Initialize / SetViewport / Update / Draw surface as the old single comet, so every comet automatically gets its own orbit polyline, particle tail, label, picking entry and HUD line. Missing/parse-error file falls back to a built-in single Halley. |

| R2 | Screen-space minimum body size | ✅ Done — `PlanetVS` expands sphere vertices outward from `uPlanetCenter` when projected radius < `uMinPixelRadius` px. |
| R3 | Adaptive star brightness | ✅ Done. New `Renderer.StarsBrightness` / `Renderer.StarsSaturation` properties drive two extra uniforms in `SkyFS` (`uBrightness`, `uSaturation`). `SolarSystemWindow.UpdateAdaptiveStars` (called once per frame before `DrawStars`) maps the camera's distance from the Sun onto a brightness lerp (`0.85` near → `0.30` deep space) and the distance to the nearest body's surface onto a saturation lerp (`1.6` hugging a planet → `0.85` empty space). The effect is barely visible at Earth (~2°) but ~90° at Neptune. Persisted via `PersistedState.LightTime`. |
| R5 | Distance ruler | Hold `Shift` while clicking two bodies to draw a labelled line ("Earth → Mars: 0.524 AU, 4.36 light-min") between them. Uses the existing label renderer. |
| R6 | Auto fit-to-orbit (`Z`) | Frame the focused body's full orbit on screen by computing the orbit's bounding sphere and lerping `Camera.Distance`. |
| R7 | Stereographic / 2D orrery mode | Top-down orthographic projection with all orbits flattened to the ecliptic. Toggle with a key. |
| R8 | Light-time marker | When R4 is on, draw a faint "echo" sphere at where the planet WAS when the photons left, plus the great-circle arc connecting echo → current position. Tiny but very cute. |
