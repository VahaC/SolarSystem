# 🌌 Solar System — Довідник можливостей

Цей документ — детальний, фіча-за-фічею, опис усього, що наразі реалізовано в симуляції Сонячної системи. Для кожної можливості наведено:

- **Що це** — короткий опис.
- **Що робить / показує** — поведінка з точки зору користувача.
- **Навіщо потрібно** — призначення фічі.
- **Як увімкнути / вимкнути** — гарячі клавіші, рядок у панелі налаштувань, CLI-параметри тощо.
- **Примітки** — деталі реалізації, збереження стану, причетні файли, особливості.

> Більшість перемикачів зберігаються у `%AppData%/SolarSystem/state.json` між запусками — інтерфейс, у якому ви вийшли, повертається при наступному запуску.

## Топові візуальні фічі

### Bloom / HDR-сяйво
- **Що це:** повноекранний пост-процес, який змушує яскраві пікселі (Сонце, спалахи, полярне сяйво, хвости частинок) «розливатися» у сусідні пікселі.
- **Що робить:** виділяє світла → 6 проходів роздільного гаусового розмиття → адитивно домішує назад до сцени.
- **Навіщо:** миттєво підіймає сприйняту якість картинки; Сонце по-справжньому *здається* гарячим.
- **Реалізація:** RGBA16F-офскрін у `Renderer` (`BeginScene` / `EndSceneAndApplyBloom`).
- **Перемикач:** панель налаштувань (`F1`) → **Bloom**.

### Плавні переходи фокусу + сліди планет
- **Що це:** при подвійному кліку або фокусі цифровою клавішею камера плавно під’їжджає до тіла замість миттєвого «тельпорту»; кожна планета лишає за собою затухаючий слід останніх позицій.
- **Що показує:** 0.5-секундний smoothstep-lerp на `Camera.Target` і `Camera.Distance` (з відстеженням актуальної позиції), та кільцевий буфер на 200 семплів на планету як `LineStrip` із градієнтом прозорості.
- **Перемикач (сліди):** `T`. Сліди очищаються при зміні напрямку часу та перемиканні масштабу.

### Логарифмічна глибина + мінімальний розмір тіл у пікселях
- **Що це:** виправлення для режиму реального масштабу (R1 / R2). Без цього астрономічне співвідношення near/far дає z-fighting, а малі планети зникають при <1 px.
- **Що робить:**
  - Лог-глибина: усі 3D-шейдери пишуть `gl_Position.z = (log2(1 + w) * Fcoef - 1) * w`, де `Fcoef = 2 / log2(far + 1)`.
  - Мінімальний розмір: `PlanetVS` радіально розширює вершини сфери відносно `uPlanetCenter`, коли проєкційний радіус < `Renderer.MinPixelRadius` (≈1 px → ≈2 px у діаметрі).
- **Перемикач:** автоматично — обидва ефекти вмикаються разом із реальним масштабом.

## Візуальне вилизування (V1–V15)

### V1 — Bloom / HDR-сяйво
Див. [Топові → Bloom](#bloom--hdr-сяйво).

### V2 — Атмосферна крайова підсвітка (rim-light)
- **Що:** френель `pow(1-dot(N,V), 3)`, тонований кольором атмосфери, для Землі, Венери, Юпітера, Нептуна.
- **Навіщо:** лімб атмосферного тіла має м’яко світитися до камери.
- **Перемикач:** автоматично на підтримуваних тілах; разом з V9.

### V3 — Шар хмар над Землею
- **Що:** друга сфера на `1.012 × VisualRadius`, альфа з яскравості текстури (`smoothstep(0.18, 0.92, lum)`), дрейфує на захід зі швидкістю `(spinRate − 0.08 об/сим-добу)`, тому шар не обертається синхронно з поверхнею.
- **Ассет:** `textures/8k_earth_clouds.jpg`. Файл відсутній → хмар немає, помилки немає.
- **Перемикач:** автоматично, якщо є текстура.

### V4 — Нічні вогні міст на Землі
- **Що:** еміссивні вогні лише з нічного боку термінатора. `PlanetFS` додає `night × (1 − smoothstep(-0.05, 0.2, NdotL))`.
- **Ассет:** `textures/8k_earth_nightmap.jpg`.
- **Перемикач:** автоматично, якщо є текстура.

### V5 — Тіні кілець Сатурна (в обох напрямках)
- **Що:** кільця кидають тінь на Сатурн, Сатурн — на кільця.
  - На освітленій півкулі Сатурна `PlanetFS` пускає промінь до Сонця, перетинає площину кілець; якщо точка попадання між внутрішнім/зовнішнім радіусами — семплить альфу текстури кілець і ослаблює дифузну + спекулярну складові на `1 − α`.
  - У `RingFS` кожен фрагмент кілець ray-marchить до Сонця проти сфери Сатурна; при попаданні фрагмент темнішає до 25%.
- **Перемикач:** автоматично для Сатурна.

### V6 — Бліки лінз (lens flare)
- **Що:** 6 кольорових «привидів» уздовж осі Сонце → центр екрана, коли Сонце приблизно у полі зору. Малюється після bloom, щоб гаусове розмиття не змазувало їх.
- **Перемикач:** автоматично; згасає за межами екрана, ховається коли Сонце за камерою.

### V7 — Покращене небо Чумацького Шляху
- **Що:** `SkyFS` відновлює напрямок огляду у світових координатах для кожного пікселя і семплить `_starsTexture` як еквіректангулярну мапу на повноекранному квадрі біля far-плейну.
- **Ассет:** `textures/8k_stars_milky_way.jpg` (фолбек: `8k_stars_milkyway.jpg`). Якщо немає — крихітна процедурна заглушка.
- **Перемикач:** автоматично.

### V8 — Затемнення та тіні тіл
- **Що:** будь-яке тіло може кидати тінь з м’яким краєм на будь-яке інше. `PlanetFS` приймає до 16 «затіняючих» сфер; кожен фрагмент тестує промінь до Сонця і застосовує `smoothstep(r·0.85, r, d)` до `diff + spec`.
- **Хто кидає:** Місяць + галілеєві + Титан + планети, обмежено 16 (`SolarSystemWindow.BuildShadowCasters`).
- **Перемикач:** автоматично.

### V9 — Атмосферне розсіювання
- **Що:** релеївська `0.75·(1+μ²)` + Henyey-Greenstein Mie (`g = 0.76`) фазові функції поверх V2-френеля, з м’яким wrap-термінатором.
- **Тіла:** Земля (холодний синій), Марс (іржа), Венера (сіркова), Титан (помаранчевий смог), Нептун (глибокий синій) — `Renderer.GetAtmosphere`.
- **Враховує затемнення:** домножується на множник з V8 — повна тінь занурює атмосферу у темряву.

### V10 — ACES-tonemapping + автоекспозиція
- **Що:** `composite.frag` застосовує ACES-фільмічну криву Кжиштофа Нарковіча до `(scene + bloom·strength) · uExposure`. При `Renderer.AutoExposureEnabled = true` 1×1 mip HDR-таргета керує авто-корекцією експозиції до сірого 0.18 (clamp 0.25–4.0).
- **Перемикач:** панель налаштувань.

### V11 — FXAA
- **Що:** `fxaa.frag.glsl` — компактний FXAA 3.x (5-tap luma-сусідство, поріг 0.0312 / 12.5%, спрямоване 4-tap розмиття з luma-range fallback).
- **Перемикач:** панель налаштувань → **FXAA**. Увімкнено за замовчуванням.

### V12 — Корона / гранули Сонця
- **Що:** `sun.frag.glsl` додає 4-октавний fbm від `normalize(vNormal)*4 + (0, t*0.04, 0)`, що задає пульсацію та гарячо-холодний tint, тож гранули рябіють по диску і не фіксуються на шві текстури.
- **Перемикач:** `U` (`Renderer.CoronaEnabled`, зберігається).

### V13 — Полярне сяйво на Землі і Юпітері
- **Що:** 128-сегментні стрічки triangle-strip між широтами 72°–80°, з хвилями завіси (`sin(angle*6 + t*0.8) + sin(angle*13 - t*1.3)`) та мерехтінням; рендеряться адитивно, тож ідуть у bloom.
- **Кольори:** Земля — холодний зелений; Юпітер — пурпурно-фіолетовий. Інтенсивність ділиться навпіл, коли сонячний вітер вимкнено.
- **Перемикач:** `K` (зберігається).

### V14 — PBR-шейдінг планет
- **Що:** Cook-Torrance з GGX (`D`), Schlick-GGX-геометрією (`G`), Schlick-френелем (`F`) над `F0 = mix(0.04, base, metallic)`. Енергозберігаюча дифузна складова через `kd = (1-F)(1-metallic)`. Параметри по тілах у `Renderer.GetPbr`.
- **Фолбек:** старий Phong при `uPbrEnabled = 0`.
- **Перемикач:** `I` (`Renderer.PbrEnabled`, зберігається).

### V15 — Спекулярна маска океану (Земля)
- **Що:** спекулярна складова множиться на червоний канал ocean-маски — блищать лише океани, континенти лишаються матовими.
- **Ассет:** `textures/8k_earth_specular_map.{png,jpg,tif}`.
- **Перемикач:** `Q` (`Renderer.OceanMaskEnabled`, зберігається).

## Симуляційні фічі (S1–S16)

### S1 — Пауза та зворотний час
- **Клавіші:**
  - `Space` — пауза / відновити.
  - `,` — назад у часі.
  - `.` — вперед у часі.
  - `+` / `-` — змінити модуль швидкості (працює в обох напрямках).

### S2 — Сліди планет
- **Що:** 200-семпловий ring buffer на планету як затухаючий `LineStrip` (альфа квадратично від віку).
- **Перемикач:** `T`. Очищаються при зміні напрямку часу та перемиканні масштабу.

### S3 — Пояс астероїдів
- **Що:** 8 000 астероїдів з попередньо обчисленими кеплерівськими елементами + перифокальний→світовий базис (з врахуванням ecliptic→GL). Щокадру кожен астероїд просувається через Newton-Raphson Kepler solve і рендериться як адитивний інстансований квад з лог-глибиною.
- **Перемикач:** панель налаштувань; кількість видно у HUD (`~`).

### S4 — Комета з іонним / пиловим хвостом
- **Що:** галлеєподібний еліпс (a≈17.83 а.о., e≈0.967, i=162°) через `OrbitalMechanics.HeliocentricPosition`, з власною лінією орбіти + CPU-частковий хвіст у конусі по антисонячній осі. Інтенсивність хвоста ∝ `1/r`, тож горить лише біля перигелію.

### S5 — Пошук дати (date seek)
- **Що:** перейти симуляційний час на конкретну дату або на дельту.
- **Клавіші:** `J` відкриває промпт зверху екрана.
  - Введіть `YYYY-MM-DD` (будь-який invariant-формат `DateTime.TryParse`) і `Enter`.
  - Або дельту зі знаком: `+30`, `-365`.
  - `Esc` — скасувати.
- **Примітка:** після стрибка сліди очищаються, щоб не лишалася стара дуга через нову епоху.

### S6 — Великі супутники
- **Що:** Іо, Європа, Ганімед, Каллісто (хост — Юпітер) і Титан (хост — Сатурн). Кожен — `Planet`, обгорнутий `Moon (hostIndex, orbitRadiusKm/artistic, periodDays, inclinationDeg, phaseDeg)`. У реальному масштабі підставляються справжні км-радіуси.
- **Кліком вибрати:** так (Q2).

### S7 — Карликові планети
- **Що:** Церера, Плутон, Хаумеа, Макемаке, Еріда з повними J2000-кеплерівськими елементами; додаються після Нептуна, тож автоматично отримують орбітну лінію, слід, picking, інфо-панель.
- **Фокус:** клік / подвійний клік (цифрові 1–8 лишаються за великими планетами).

### S8 — Оверлей сузір’їв
- **Що:** RA/Dec-точки ліній з `data/constellations.json`, малюються як скайбокс на небесній сфері в нескінченності (з матриці виду прибрано трансляцію + `gl_Position.z = w`). Назви — через `BitmapFont`, прив’язані до `camera.Eye + dir * R`.
- **У комплекті:** Оріон, Велика Ведмедиця, Кассіопея, Лебідь, Ліра, Південний Хрест, Скорпіон, Лев.
- **Перемикач:** `C`.

### S9 — Космічні апарати та зонди
- **Що:** Voyager 1, Voyager 2, JWST, ISS — як адитивні 3-осьові «хрестики» з підписами.
  - Voyager: фіксований напрямок втечі + лінійний дрейф а.о./рік від запуску.
  - JWST: припаркований антисонячно від Землі на відстані Sun-Earth L2.
  - ISS: 92.68-хвилинна кругова орбіта Землі на 6 778 км (реал-скейл) або `Earth.VisualRadius * 1.3` (компресовано).
- **Перемикач:** `P`.

### S10 — Точки Лагранжа
- **Що:** L1..L5 щокадру для пар Sun–Earth і Sun–Jupiter за аналітичними CR3BP-наближеннями: `R · (μ/3)^(1/3)` для L1/L2, `R · (1 + 5μ/12)` для L3, рівносторонні апекси для L4/L5.
- **Малюється:** маленький адитивний ромб + підпис.
- **Перемикач:** `G`.

### S11 — Метеорні потоки
- **Що:** 7 максимумів (Квадрантиди, Ліриди, η-Аквариди, Персеїди, Оріоніди, Леоніди, Геміни́ди) як `(місяць, день, RA/Dec радіанта, частота)`. У ±3 дні від піка з диска перпендикулярного до радіанта поряд із Землею вилітають короткі адитивні «доріжки».
- **Банер:** знизу справа називає активний потік.
- **Перемикач:** `M`.

### S12 — Календар затемнень / транзитів
- **Що:** ручний список визначних Sun–Earth–Moon вирівнювань і транзитів Меркурія / Венери 1999–2045 (`Bookmarks`).
- **Клавіші:**
  - `Ctrl+B` — наступний bookmark після поточного `simDays` (з закільцюванням).
  - `Ctrl+Shift+B` — попередній (Q8).
- **Примітка:** сліди очищаються, банер показує назву події.

### S13 — Візуалізація приливного захоплення
- **Що:** маленька адитивна 3-сегментна стрілка на кожному супутнику з синхронним обертанням (Місяць, галілеєві, Титан) — указує на хост-планету.
- **Перемикач:** `F4` (`PersistedState.ShowTidalLock`, зберігається).

### S14 — Індикатор планетарного вирівнювання
- **Що:** для кожної великої планети рахує геліоцентричну екліптичну довготу, виконує union-find з порогом 12° і повертає кожну компоненту розміру ≥3 як `Group(indices, names)`. Малює адитивний промінь від Сонця через усіх членів.
- **Банер:** зверху справа з учасниками.
- **Перемикач:** `F5`.

### S15 — Режим N-тіл (з збуреннями)
- **Що:** velocity-Verlet (kick-drift-kick leapfrog) для великих планет в одиницях (а.о., доба, M⊙) з `GM_sun = 4π²/365.25² ≈ 2.959e-4 а.о.³/доба²`. Сонце нерухоме на початку координат; взаємна гравітація використовує табличні маси. Адаптивний sub-step обмежує `|dt|` на 0.5 доби/крок (макс 200 кроків/кадр). При вмиканні / стрибку дати / скрабі — ре-синк з аналітичної кеплерівської позиції + швидкості 1-добовою центральною різницею.
- **Банер:** «N-body mode» поки активно.
- **Перемикач:** `F6` (зберігається).
- **Примітка:** карлики / супутники / комети лишаються аналітичними — у цьому весь сенс контрасту.

### S16 — Каталог реальних комет
- **Що:** JSON-каталог (`data/comets.json`) — Галлей, Хейл-Бопп, NEOWISE, Енке — з повними кеплерівськими елементами + індивідуальним налаштуванням хвоста (`emissionRate`, `tailLifetime`, `tailSpeed`). Кожна отримує свою лінію орбіти, частковий хвіст, підпис, picking, рядок у HUD.
- **Фолбек:** немає файлу / помилка парсингу → вбудований одиничний Галлей.

## Зручність користування (Q1–Q15)

### Q1 — Плавні переходи фокусу
Див. [Топові → Плавні переходи фокусу](#плавні-переходи-фокусу--сліди-планет).

### Q2 — Клік по супутниках/кометах
- **Що:** Місяць + галілеєві + Титан + Галлей лежать у `_extraBodies`. `TryPick` проектує їх разом з планетами, тож клік-інфо, подвійний клік-фокус, плавні переходи й `ToggleRealScale` zoom-fit працюють і для них.

### Q3 — Пошук тіл за назвою
- **Клавіші:** `Ctrl+F` відкриває модальний промпт зверху; набір тексту фільтрує (case-insensitive, спочатку префікс, потім substring) Сонце + планети + карлики + Місяць + великі супутники + комету. Топ-5 кандидатів попереду.
  - `Enter` — фокус на кращому збігу.
  - `Esc` — скасувати.

### Q4 — Скріншот (Windows-варіант)
- **Клавіша:** `F12`. Зберігається в `screenshots/screenshot_yyyyMMdd_HHmmss.png`.
- **Примітка:** на не-Windows перекритий кросплатформною Q11.

### Q5 — Збереження налаштувань
- **Що:** `%AppData%/SolarSystem/state.json` round-trip POCO `PersistedState` через `System.Text.Json`.
- **Зберігається:** камера (yaw/pitch/distance/target), `_simDays`, `_daysPerSecond`, `_paused`, `_focusIndex`, усі UI-перемикачі, прапори solar-wind / flares, `OrbitalMechanics.RealScale`.
- **Життєвий цикл:** завантаження в кінці `OnLoad` (після збору світу); збереження першим у `OnUnload`. RealScale застосовується перший через `ToggleRealScale`, щоб дистанція клампилася під правильні межі.

### Q6 — Тултип при наведенні мишею
- **Що:** раз на кадр `OnUpdateFrame` робить `TryPick(_mousePos)`; при попаданні малюється маленький `Назва\n0.000 а.о.` поряд з курсором.
- **Приховується:** під час date-seek / name-search промптів.

### Q7 — Оверлей FPS / лічильника частинок
- **Клавіша:** `~` (`Keys.GraveAccent`). FPS згладжується на вікні 0.5 с. Показує: FPS, режим масштабу, лічильники wind / flare / comet-tail (`Active / Max`), кількість астероїдів.

### Q8 — Закладки
- **Що:** читає `data/bookmarks.json` (`System.Text.Json`, коментарі та хвостові коми дозволені), фолбек на вбудований каталог.
- **Клавіші:** `Ctrl+B` — наступна, `Ctrl+Shift+B` — попередня.
- **Побічно:** очищає сліди, грає Q15-«тік», банер `kind: title — yyyy-MM-dd`.

### Q9 — Скрабер таймлайна
- **Клавіша:** `V` — увімкнути/вимкнути. 60-клітинна `█/░` смуга внизу мапить mouse-X на ±100 років навколо J2000. Поки тягнеш каретку, звичайне `_simDays += daysPerSecond·dt` придушується; сліди очищаються при стрибку >0.5 доби.

### Q10 — Кінематичні шляхи камери
- **Що:** 9 слотів-вейпойнтів, кожен зберігає `Yaw / Pitch / Distance / Target`. Програвання — Catmull-Rom-сплайн на 4 точки впродовж 6 с (позиція, yaw, pitch, distance інтерполюються незалежно).
- **Клавіші:**
  - `Ctrl+1..9` — записати / перезаписати.
  - `Ctrl+Shift+1..9` — стерти один слот.
  - `Shift+P` — програти.
  - `Ctrl+Shift+P` — стерти всі.
- **Збереження:** `%AppData%/SolarSystem/campath.json`. Поки йде програвання, гілка «слідувати за фокусованим тілом» пропускається.

### Q11 — Кросплатформний скріншот
- **Що:** `SaveScreenshot` тепер на `glReadPixels` RGBA8 + перевертання рядків + SkiaSharp `SKBitmap` + PNG якості 95.
- **Клавіша:** `F12` (більше не лише Windows).

### Q12 — Внутрішня панель налаштувань
- **Клавіша:** `F1`. Власноруч написаний «ImGui-lite» оверлей через `Renderer.DrawText` — без додаткового GL-стану. 17 рядків-перемикачів + слайдер швидкості, усе через замикання над тими ж полями, що й клавіатурні шорткати, тож вони синхронні.
- **Миша:** кліки тестуються по bbox’ах рядків і поглинають LMB, щоб камера не «смикалась».
- **Адаптивна висота:** панель кламиться по висоті екрана, а коли всі рядки не вміщуються — мишеве колесо (наведене на панель) прокручує список. Поза екраном опиняються лише невидимі рядки; їхні hit-box’и обнуляються, тож «сліпого» кліку по схованому рядку не буде.

### Q13 — Локалізація
- **Що:** `Localization.T(key)` над пласким `Dictionary<string,string>`. Англійські дефолти зашиті в код; рантайм-мови — `data/lang.<code>.json` (наприклад `lang.uk.json`). При старті береться системна культура, якщо є відповідний файл.
- **Клавіша:** `F2` циклічно перемикає всі знайдені мови. Зберігається в `state.json`.

### Q14 — Згортання довідки
- **Клавіша:** `Tab` циклить `_helpMode`:
  - `0` — повна Controls-панель + інфо знизу зліва.
  - `1` — лише дата + швидкість зверху.
  - `2` — все приховано.
- **Адаптивна розкладка:** список Controls тепер автоматично розкладається у кілька колонок, а за потреби ще й зменшує шрифт (≥8 px), щоб уся шпаргалка вміщалась навіть на 1280×720. Кількість колонок підбирається так, щоб усі 51 рядків влізли по висоті, після чого ширина колонки клампиться під ширину вікна.
- **Підказка-пошуковик:** у режимі `1` під рядком швидкості з’являється сіра підказка «Tab — довідка · F1 — налаштування · F3 — закладки · Ctrl+F — пошук», щоб користувач знайшов меню навіть коли повний список схований.
- **Збереження:** в `state.json`.

### Q15 — Звуки / Mute
- **Що:** `AudioService` грає «whoosh» на `BeginFocusTransition` і «tick» на стрибок дати / закладку / запис вейпойнта / скріншот.
- **Реалізація:** на Windows — `Console.Beep` у `Task` (щоб не блокувати головний потік); на Linux/macOS — мовчазний no-op, перемикач лишається користувачу.
- **Клавіша:** `S` (зберігається).

## UX режиму реального масштабу (R1–R8)

> Сам режим реального масштабу перемикається `R`. Це перемикає `OrbitalMechanics.RealScale`, підставляє реальні км-радіуси / орбіти супутників і застосовує клампи дистанції камери для цього режиму.

### R1 — Логарифмічна глибина
Див. [Топові → Лог-глибина + мінімальний розмір](#логарифмічна-глибина--мінімальний-розмір-тіл-у-пікселях).

### R2 — Мінімальний екранний розмір тіла
Те саме. `PlanetVS` радіально розширює вершини сфери, коли проєкційний радіус < `uMinPixelRadius`.

### R3 — Адаптивна яскравість зір
- **Що:** `Renderer.StarsBrightness` і `Renderer.StarsSaturation` ідуть у `SkyFS`. `SolarSystemWindow.UpdateAdaptiveStars` (раз на кадр перед `DrawStars`) мапить:
  - дистанцію камери від Сонця → яскравість `0.85` (близько) → `0.30` (далекий космос).
  - дистанцію до поверхні найближчого тіла → насиченість `1.6` (впритул до планети) → `0.85` (порожній простір).
- **Перемикач:** автоматично; референсні радіуси перемикаються між реал/компресованим масштабом, щоб «відчуття» лишалося.

### R4 — Візуалізація часу проходження світла
- **Що:** кут спіну планети (і її хмарного шару) обчислюється на `simDays - r/c`, а не `simDays`, де `r` — геліоцентрична відстань, `c = 173.1446 а.о./добу`. Позиції тіл лишаються поточними; зсувається лише освітлена довгота.
- **Видимий ефект:** ~2° на Землі, ~90° на Нептуні.
- **Клавіша:** `Y` (зберігається).

### R5 — Лінійка відстаней *(заплановано)*
- **Що:** з утриманим `Shift` клікаєш по двох тілах — малюється підписана лінія («Земля → Марс: 0.524 а.о., 4.36 світлохвилин»).
- **Статус:** roadmap.

### R6 — Auto fit-to-orbit *(заплановано)*
- **Клавіша:** `Z` — обрамити повну орбіту фокусованого тіла на екрані.
- **Статус:** roadmap.

### R7 — 2D-orrery (стереографічний) режим *(заплановано)*
- **Що:** ортогональна проєкція згори, всі орбіти спроєктовані на екліптику.
- **Статус:** roadmap.

### R8 — Маркер ехо світлового часу *(заплановано)*
- **Що:** при увімкненому R4 малювати тьмяну «ехо»-сферу там, де планета БУЛА, коли світло стартувало + дугу великого кола ехо → поточне положення.
- **Статус:** roadmap.

## Архітектура / інструментарій (A1–A12)

### A1 — Кросплатформні шрифти
- **Що:** `BitmapFont` на SkiaSharp 2.88.8 (з `SkiaSharp.NativeAssets.Linux.NoDependencies` для Linux). Ланцюг typeface-фолбеків: `Segoe UI` → `DejaVu Sans` → `Arial` → `SKTypeface.Default`. `SubpixelAntialias`. Гліфи малюються у scratch-`SKBitmap`, ink-box знаходиться по альфа-покриттю, блітається в RGBA8888-атлас. Публічний інтерфейс `Glyph` той самий, що й до рефакторингу.

### A2 — Шейдери у `.glsl`-файлах
- **Де:** `Resources/Shaders/*.glsl`, копіюються через `<None CopyToOutputDirectory="PreserveNewest">`.
- **Хелпер:** `ShaderSources.Load(name)` / `CreateProgram(vs, fs)` з кешем; розв’язує шляхи відносно `AppContext.BaseDirectory` із фолбеком на CWD.

### A3 — Дані планет у `planets.json`
- **Що:** `Planet.CreateAll()` / `CreateDwarfPlanets()` спочатку пробують `data/planets.json`, при відсутності/помилці — вбудована таблиця. Коментарі та хвостові коми дозволені.

### A4 — Інстансовані квад-частинки
- **Що:** `InstancedQuadParticles` тримає 4-вершинний квад-VBO + per-system динамічний instance-VBO `vec4(pos.xyz, life01)`, через `glVertexAttribDivisor`, `glDrawArraysInstanced(TriangleStrip, 0, 4, count)`. Використовують `SolarWind`, `SolarFlares`, `Comet` (хвіст), `AsteroidBelt`. `particle.vert` відтворює стару криву `gl_PointSize` у clip-space, тож розмір квадів не залежить від драйвера.

### A5 — Незалежні від частоти кадрів частинки
- **Що:** динамічні системи (`SolarWind`, `SolarFlares`, `Comet`) розбивають `dt` на `ceil(dt / MaxSubStep)` фіксованих суб-кроків (`MaxSubStep = 1/60 с`, не більше 16). Високий `_daysPerSecond` більше не «тіниться» (alias) у рух частинок чи таймери burst’ів. `AsteroidBelt` аналітичний — не зачіпається.

### A6 — Гарячий перевант шейдерів (GLSL hot-reload)
- **Клавіша:** `F7` — увімкнути/вимкнути `FileSystemWatcher` над `Resources/Shaders/*.glsl`. Дискові події коалесуються в потокобезпечну чергу; `OnUpdateFrame` раз на кадр викликає `ShaderSources.PollPendingReloads` у GL-потоці. `ShaderProgram.Reload` спочатку лінкує нову програму як тестову і лише при успіху міняє `Handle` (та чистить кеш uniform-локацій) — описка в шейдері лишає попередню програму живою, помилка з’являється у банері.

### A7 — Headless-рендер / експорт відео
- **CLI:** `--render --from YYYY-MM-DD --to YYYY-MM-DD [--dt 1.0] [--frames N] [--fps 60] [--out render] [--ffmpeg path] [--video-out file.mp4] [--real-scale]`.
- **Поведінка:** `StartVisible = false`, persisted state не чіпається, sim-час фіксований на `From + FrameIndex * dt` (детермінований кадр), частинкам — фіксований `1/Fps` суб-крок, кожен `SwapBuffers` зберігається у `OutDir/frame_NNNNN.png` через `SaveScreenshotTo`. Після останнього кадру викликається ffmpeg: `ffmpeg -y -framerate Fps -i frame_%05d.png -c:v libx264 -pix_fmt yuv420p -crf 18 out.mp4`, після чого вікно закривається і процес виходить.

### A8 — Compute-shader N-body *(заплановано)*
- Перенести Kepler-solve астероїдів у compute-шейдер; стеля 8 000 → 100 000+. Вихід прямо в instance VBO.

### A9 — Юніт-тести `OrbitalMechanics` *(заплановано)*
- xUnit + GitHub Actions. Перевірка `SolveKepler`, `HeliocentricPosition`, `OrbitWorldScale` проти канонічних J2000-ефемерид.

### A10 — CI smoke-build *(заплановано)*
- Матриця GitHub Actions `windows-latest` / `ubuntu-latest` / `macos-latest` із `dotnet build -c Release`.

### A11 — Native AOT *(заплановано)*
- Trim + AOT-publish на .NET 10. Прибрати reflection-залежні шматки, перевести `System.Text.Json` на source generators.

### A12 — Профайлер-оверлей на кадр *(заплановано)*
- Розширити `~`-HUD кумулятивними GPU + CPU таймінгами на пасок (sky, planets, particles, bloom) через `GL_TIMESTAMP` queries.
